## 핵심 내용

- 사실: 포스트는 배너 제작 업무가 규칙이 명확하고 반복적이어서 생성형 AI에 위임하기 적합하다고 설명한다.
- 사실: 작성자는 ChatGPT 단일 프롬프트만으로는 서비스 컨테이너 사양, 디자인 톤, Admin API 형식 등 맥락 부족 때문에 실사용이 어려웠다고 밝혔다.
- 사실: 이를 SKILL.md에 역할, 제약, 도구를 정의하는 방식으로 보완했다고 설명한다.
- 의견: 저자는 프롬프트보다 SKILL.md + 레퍼런스 폴더가 더 시스템처럼 동작한다고 주장한다.

## 시사점

- AX 관점에서는 생성형 AI 자동화의 성공 조건이 모델 자체보다 업무 규칙의 명시화와 실행 환경 연결에 있음을 보여준다.
- 기존 지식 베이스의 **생성형 AI의 기업 활용 사례** 및 **생성형 AI·AI Agent 기반 업무 프로세스 혁신과 조직 AX 실행**과 연결된다. 단발성 생성에서 반복 가능한 운영 프로세스 설계로 이동하는 흐름의 구체 사례다.
