클로드 코드를 전사 도입한 지 한 달이 지났습니다
변화의 속도가 생각보다 훨씬 빨랐습니다. 처음에는 다들 비슷했던 것 같습니다. AI를 문장 다듬기나 초안 작성처럼, 개인 생산성을 조금 높여주는 도구 정도로 받아들이는 경우가 많았습니다. 그런데 한 달이 지나고 보니, Claude Code는 생각보다 다른 위치로 이동하고 있었습니다.
1. 생각보다 어렵지 않았습니다.
이번 타운홀에서 인상적이었던 점 중 하나는, Claude Code가 생각만큼 소수의 숙련자만 쓰는 도구는 아니라는 점이었습니다. 처음에는 다소 어렵게 느껴질 수 있지만, 실제로는 각 팀이 자신들의 반복 업무나 불편한 작업을 하나씩 자동화하는 방식으로 빠르게 적응하고 있었습니다. “AI를 잘 쓰는 몇 명”의 이야기가 아니라, 팀 단위로 써보기 시작한 도구에 가까워 보였습니다.
2. 생각보다 빨리 ‘쓰기’에서 ‘만들기’로 넘어가고 있었습니다.
CSM팀은 콜 로그, 미팅 로그, 결제 더블체크처럼 반복적으로 들어가는 업무를 자동화하기 시작했습니다. 예전에는 문장을 다듬거나 초안을 만드는 정도였다면, 이제는 실제 업무 흐름을 줄이고 바꾸는 방향으로 활용이 확장되고 있었습니다.
CX팀의 발표도 인상적이었습니다. 사용가이드 자동화, FAQ 자동 생성, TASK 자동 생성, 서류 자동 발급, 온보딩과 평가 보조까지, 지난 한 달 동안 여러 앱을 직접 만들었습니다. 단순히 생산성을 높이는 도구를 쓰는 것을 넘어, 각 팀이 자신의 문제를 해결하는 도구를 직접 만들기 시작한 것입니다.
3. 어쩌면 가장 큰 변화는 생산성이 아니라 주도권이었습니다.
예전에는 팀 안에서 반복적으로 생기는 불편이나 비효율이 있어도, 보통은 참고 쓰거나 개발을 기다리거나 사람을 더 투입하는 방식으로 풀었습니다. 그런데 이번 타운홀에서 보인 건 다른 장면이었습니다. 각 팀이 자신들의 문제를 직접 자동화하고, 직접 앱으로 만들고, 직접 해결 가능한 형태로 바꾸기 시작했습니다. Claude Code는 일을 조금 빨리 하게 만드는 도구라기보다, 문제 해결의 주도권을 팀 안으로 다시 가져오는 도구에 가까워 보였습니다.
한 달 전에는 Claude Code가 개인의 생산성을 조금 높여주는 도구처럼 보였는데, 지금은 팀의 일하는 방식을 실제로 바꾸기 시작한 도구에 가까워 보입니다.
생각보다 어렵지 않았고, 생각보다 훨씬 빠르게 변화를 만들고 있었습니다.
