## 원문

🎉 Amazon S3 Files — S3를 파일 시스템처럼 사용하는 새로운 스토리지 패러다임!
⌚ 발표일: 2026년 4월 7일
Amazon S3가 S3 Files 기능을 통해 파일 시스템 인터페이스를 공식 지원합니다.
이제 S3 데이터를 객체(Object)가 아닌 파일 시스템(File System)처럼 직접 접근할 수 있습니다.
✨ 주요 특징
1️⃣ S3를 완전한 파일 시스템으로 사용
- POSIX 스타일 파일 시스템 semantics 제공
- 기존 파일 기반 애플리케이션 그대로 사용 가능
- 코드 변경 없이 S3 데이터 접근
- Object Storage + File System 통합
2️⃣ 데이터 이동 없이 직접 접근
- 기존: S3 ↔ 파일 시스템 간 데이터 복사 필요
- 신규 : S3 데이터 그대로 파일처럼 접근을 통해 데이터 중복 제거
3️⃣ EFS 기반 고성능 아키텍처
- Amazon Elastic File System 기반 설계
- 낮은 지연 시간 + 높은 처리량 제공
- 캐싱을 통한 빠른 데이터 접근으로 고성능 파일 시스템 경험
4️⃣ Object + File 동시 접근
- S3 API와 파일 시스템 접근 동시 지원
- 동일 데이터를 두 방식으로 활용 가능으로 유연한 데이터 활용
5️⃣ 대규모 병렬 접근 지원
- 수천 개 컴퓨트 리소스 동시 연결
- EC2 / 컨테이너 / 서버리스 환경 지원으로 대규모 워크로드 최적화
6️⃣ AI/ML 및 데이터 파이프라인 최적화
- 데이터 레이크 직접 처리 및 ML 데이터 준비 작업
- AI 에이전트 상태 공유 및 분석 워크로드
- 데이터 파이프라인 단순화
7️⃣ 초고성능 및 확장성
- 액티브 데이터 캐싱
- 수 TB/s 수준 집계 읽기 처리량
- 스토리지 성능 병목 제거로 대규모 데이터 처리 최적화
8️⃣ 마이그레이션 없이 즉시 사용
- 기존 S3 버킷 그대로 사용
- 별도 데이터 이동 불필요하기 때문에 즉시 도입 가능
