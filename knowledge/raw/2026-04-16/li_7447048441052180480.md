과제 7개가 제출되면,
링크드인 초안 7개가 자동으로 생성됩니다.
매주 과제를 제출하고, 서로의 과정을 공유하며 성장하는 구조로 운영하고 있어요.
—
근데 3주차까지 노션으로 아카이빙하다 보니 한 가지 문제가 생겼어요.
과제가 쌓이는데 활용이 안 되는 거예요.
누가 어떤 도구를 썼는지, 주차별로 어떤 변화가 있었는지,
데이터로 뽑아내려면 해당 노션을 익스포트해서
LLM에 넣어주는 귀찮은 작업이 필요했거든요.
즉, 기록은 쌓이는데 인사이트가 바로 안 나오는 구조.
—
그래서 4주차부터 PM 김다솔 을 필두로
옵시디언 + GitHub으로 전환했습니다.
핵심은 이거예요.
과제 데이터를 '보관'이 아니라 '데이터베이스'로 쓰겠다는 것.
과제를 MD 파일로 작성 → GitHub 저장소에 푸시
옵시디언 터미널 플러그인으로 Claude Code 직접 실행
문 저장소를 소스로 팀 웹사이트 자동 생성 (갤러리 뷰 + AI 분석)
주 주차별 과제를 기반으로 링크드인 초안까지 자동 생성
이번 주 7명의 과제가 올라오자마자,
각자의 핵심 주제에 맞는 링크드인 초안 7개가 한 번에 만들어졌어요.
—
노션일 때는 "기록했다"에서 끝났는데,
옵시디언으로 바꾸니까 "기록이 콘텐츠가 되고, 분석이 되고, 브랜딩이 되는" 구조가 됐어요.
도구를 바꾼 게 아니라 데이터가 흐르는 방식을 바꾼 거예요.
팀의 기록이 노션에 쌓여만 있다면,
