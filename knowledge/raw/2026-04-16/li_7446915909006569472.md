그걸 해결하는 게 CLAUDE.md입니다.
CLAUDE.md를 제대로 쓰는 법, 12가지로 정리했습니다.
① CLAUDE.md = Claude에게 주는 지속적인 지침 파일
매 세션 시작 시 자동으로 로드됩니다.
한 번 잘 써두면, 매번 설명할 필요가 없어집니다.
② 파일 위치가 범위를 결정합니다
관리 정책: /etc/claude-code/ → 조직 전체 강제 적용
프로젝트: ./CLAUDE.md → Git으로 팀 공유
사용자: ~/.claude/CLAUDE.md → 본인만, 모든 프로젝트
→ 더 구체적인 위치가 우선 적용됩니다.
③ 이렇게 쓰면 Claude가 무시합니다
❌ "코드를 제대로 포맷합니다" ✅ "2칸 들여쓰기 사용"
❌ "변경사항을 테스트합니다" ✅ "커밋 전 npm test 실행"
검증 가능할 만큼 구체적으로 쓰세요.
④ 파일당 200줄 이하를 지키세요
길수록 Claude의 준수율이 떨어집니다.
내용이 많아지면 @import로 파일을 분리하거나 .
claude/rules/ 로 쪼개세요.
⑤ @import로 외부 파일을 불러올 수 있습니다
@README @package.json @~/.claude/my-preferences.md
Git에 올리기 싫은 개인 설정은 홈 디렉토리에서 불러오면 됩니다.
⑥ .claude/rules/ 로 규칙을 쪼개세요
code-style.md / testing.md / security.md
주제별로 파일을 나누면 팀이 유지 관리하기 훨씬 쉬워집니다.
⑦ 경로별 규칙으로 컨텍스트를 아끼세요
paths: "src/api/**/*.ts"
이렇게 설정하면 해당 파일을 열 때만 규칙이 로드됩니다.
항상 켜두지 않아도 됩니다.
⑧ 자동 메모리는 Claude가 스스로 쓰는 노트입니다
CLAUDE.md는 내가 씁니다. 자동 메모리는 Claude가 씁니다.
"항상 pnpm 써" → Claude가 알아서 기억합니다.
세션이 끊겨도 다음에 기억하고 있습니다.
⑨ 로드 순서가 있습니다
관리 정책 → 사용자 → 상위 디렉토리 → 프로젝트 → 하위 디렉토리
모노레포에서 다른 팀 CLAUDE.md가 섞이면
claudeMdExcludes 설정으로 제외할 수 있습니다.
⑩ /memory 명령을 습관으로 만드세요
현재 세션에 어떤 파일이 로드됐는지 바로 확인됩니다.
파일이 목록에 없으면 Claude는 볼 수 없습니다.
쓰고 나서 반드시 확인하세요.
⑪ /compact 후 지침이 사라졌다면
대화에만 있던 지침은 compact 후 소실됩니다.
