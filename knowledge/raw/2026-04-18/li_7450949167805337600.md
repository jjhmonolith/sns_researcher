## 원문

대한민국 법령 Git과 MCP 사용해보기 #1
행정규칙(고시) 내 별표 자료 조회에서 어려움이 생겼습니다. [별표]도 쉽게 가져오는 팁이 있을까요..?
요즘 업무 장면 중 손이 많이 가는 하나를 골라, 깃과 MCP 모두 테스트 해보고 있는데요. 행정규칙(고시) 안에 있는 [별표]자료 조회에서 막혀있습니다.
// 업무상황 //
“모성보호 관련 급여 및 지원금 내용을 짧은 시간 내 정리하고, 효율적으로 업데이트 하고싶다!(정확도는 무조건 필수)”
정부 지원 급여/지원금이 많지만 정보가 산재되어 있고, 개정도 잦아 매번 찾아보기가 까다롭습니다. 특히 최근 출산육아와 관련된 제도가 자주 개정되고 있어요.
그래서 모성보호관련 급여,지원금을 깃과 mcp로 쉽게 정리할 수 있는지 테스트 해봤습니다.
// 프롬프트 //
모성보호관련 지원금, 급여를 찾아줘. 지원대상은 근로자, 사업주 모두를 포함하고, 명칭, 지원조건, 금액, 시행일, 개정 전 금액 비교, 근거법령(법률,시행령,시행규칙,고시), 근거법령 링크를 정리해줘.
// 테스트 //
ㅣ1ㅣ 깃 (legalize-kr)
▶️ 사용 환경 : 로컬에 클론 후, vs code, claude cowork에서 사용
▶️ 특징
▪️ 체감상 MCP보다 좀 더 빠른 듯 했음
▪️ 현재 법률, 시행령, 시행규칙(부령), 기타 대통령령까지만 다루고 있어, 고시 정보가 없음
ㅣ2ㅣ MCP (korean-law-mcp)
▶️ 사용 방법 : 클로드에 custom connector 추가 하고, cowork에서 사용
▶️ 특징
▪️고시 내 [별표]에 있는 정보까지 검색이 되긴 함. 다만, 근거법령 원문 링크를 같이 달아달라고 했는데, 고시는 연결 안되는 링크가 붙기도 함
▪️ 내가 필요한 정보는 [별표5]에 있는데, 법제처 API 50,000자 제한으로 조회가 불가능하다고 함
▪️ API로 안되니 웹서치 해오는데, 조금씩 최신 정보가 아닌 부분이 있음
▪️ 전반적으로 정보 정리를 잘 해줌. 별표 자료 조회만 잘 된다면 프롬프트 다듬어서 바로 사용 가능
ㅣ3ㅣ추가 : claude in chrome 사용
▪️ 클로드가 claude in chrome로 [별표5]를 읽을 수 있다고 사용해보라고 함
▪️정보를 찾아오긴 함. 오래걸림.. 이 시간이면 직접 링크 들어가서 보는게 빠를 것 같음
▪️ 스스로 엔드포인트 찾고, 별표 파일 OCR 했다가, 뭐가 잘 안되는지 이미지 파일로 저장해서 청크 단위로 base64 추출해서 조합한다고 함
▪️ [별표5] 안에 있는 표 정보는 또 못 가져옴. 그거 다시 인식해달라고 했더니… 토큰 모두 소진되어 중단됨
// 생각 정리 //
▪️클로드에서 하면 워드 형식이나, html로 정리해주니, 누구나 업무에 활용하기 편리할 것 같음
▪️ 세부 정보가 [별표]에 기재된 경우가 많고 그 안에 또 표가 있어서, 이를 어떻게 해결할 수 있을지 찾아 보고자 함
▪️정보 조회가 원활히 되면, 프롬프트를 더 다듬어봐야겠음
▪️ [별표]자료는 원문 링크만 붙여두고, 사람이 확인해야할까요…?
