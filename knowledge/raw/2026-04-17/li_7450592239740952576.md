Since early this month, the idea of a "Second Brain" for AI agents has been blowing up. Andrej Karpathy released LLM-Wiki, Garry shared GBrain, and everyone started talking about giving agents real, persistent memory.
And we'd already built it as a working product. Thanks to Garry's repost, we saw a big spike. 400 new users signed up. Our X account crossed 1K followers. We turned on payments today, and 15 people already subscribed. Discord just passed 700 members.
But we didn't start last week. We've been building this for four months. The future we saw was very simple. In a world full of agents, the bottleneck won't be models or tools. It'll be memory. And that memory can't live in a silo. It needs to be shared. Think about how you grab a doc from Google Drive when you need it for work. Now imagine every agent you use pulling from a Second Brain that actually knows you. What to do, how to do it, and what context matters.
When we looked at the market, we found two problems. Most memory products are B2B SDK or too buggy. If you've tried building a memory system with Mem0, Zep, or Neo4j, you know what I mean. Powerful, but way too complex for regular people.
We wanted something different. We wanted everyone to have their own Second Brain. So we built the most seamless memory layer we could. No setup.
You connect what you already use: agent conversations, messengers, calendar, email, Obsidian, Notion. Membase organizes it all automatically, links related knowledge, and makes it available to your agents. Every agent, no matter where it runs, shares the same context. No more 40-line prompts. No more attaching files every time. It just knows.
We're in private beta right now. If you're interested, drop a comment and I'll DM you an invite code with a free month of Pro.
