# 강보라: AI가 데이터 추출·분석 실행의 장벽을 낮추면서 분석가의 역할이 ‘실행’에서 ‘결정(맥락·한계 고려)’으로 이동한다고 주장합니다.

**AI 요약**: AI가 데이터 추출·분석 실행의 장벽을 낮추면서 분석가의 역할이 ‘실행’에서 ‘결정(맥락·한계 고려)’으로 이동한다고 주장합니다.

**게시일**: 2026-04-11

**참신성 점수**: 45/100

**참신성 이유**: 데이터 분석가 역할 변화에 대한 관점은 있으나, Mixpanel MCP 등 구체적 실사용 결과/수치 없이 일반적 논지로 전개됩니다.

**토픽**: 기업 AI 도입/전환 관점, AI로 인한 데이터 분석 업무 프로세스 변화, AI 거버넌스/책임(결정·맥락·한계 고려)


## 원문

“너무 좋네. 이제 데이터 분석가는 끝났군”
얼마전 데이터 분석가로 오래 일해온 선배가 Mixpanel MCP를 써보시더니 이리 말씀하시더라구요. 정말 데이터 분석가는 끝났을까요?
매일 기업의 분석 툴과 데이터 분석을 다루는 입장에서 AI가 '빨리', 심지어 '잘' 하는 걸 보며 놀라는게 하루이틀일은 아니지만 주변분들의 걱정을 들으면 다시금 두려워지고는 합니다.
———
그런데 생각해보면 이 상황이 낯설지는 않습니다.
포스트 SQL 분석툴들이 쏟아져 나올때 한 약속이 이제야 실현된 것 뿐이거든요.
“SQL 몰라도 누구나 데이터를 볼 수 있게 해드립니다.”하는 약속.
다만 지금까지는 그 약속에 조건이 붙어 있었습니다.
“대신 이 툴 사용법은 익히셔야 해요 (SQL보다는 쉬움. 찡긋)”
사용자는 드래그앤드롭이든, 클릭이건, 결국 실행을 위한 기술은 배워야 했죠.
AI는 그저 기존의 툴 학습 이행 조건을 없애고
'누구나, 아무 학습 없이, 데이터를 꺼낼 수 있게 한다'는
오래된 약속을 실현한 것 뿐입니다.
———
다시 돌아오면 결론적으로 데이터 분석은 끝나지 않았다고 봅니다.
회사는 선택을 하고 책임질 사람이 필요한 곳이고,
그러기에 AI가 산출한 결과에서 어떤 숫자를 고를지,
무슨 맥락과 한계를 고려해 어떤 선택과 책임을 질지는 사람이 해야하는 일이니까요.
AI가 격파해준 데이터 추출 실행의 장벽은 관문일 뿐,
그 뒤에는 커다란 결정의 장벽이 기다리고 있습니다.
현 시대의 분석가들이 '데이터는 이런데 그래서 어떻게 해야하나' 고민하던 곳이죠.
AI는 실행의 장벽을 넘어 맥락을 가진 수 많은 사람들을 결정의 장벽앞에 데려다 놓았을 뿐입니다.
데이터 풍요와 해석의 풍요에서 어떤 한계와 이점을 고려해 결정할지를 돕는게
앞으로의 데이터 분석의 역할이 되지 않을까요?
분석가의 총 수는 줄어들 수 있겠지만 너비는 더 넓어졌다고 감히 판단해봅니다.
여러분의 생각도 궁금합니다.
