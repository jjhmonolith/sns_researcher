## 원문

배너 하나 만드는 데 15분이 걸렸습니다.
그라데이션 고르고, 반응형 3개 breakpoint 맞추고, 클래스명 충돌 안 나게 접두어 붙이고, WCAG 대비율 확인하고, Admin API로 등록하고.
창의적인 작업이 아닙니다. 규칙이 명확하고, 체크리스트가 있고, 반복됩니다.
그래서 AI에게 맡겼습니다.
처음엔 ChatGPT에 "배너 HTML 만들어줘"라고 해봤습니다.
결과물은 나왔지만 쓸 수 없었습니다.
우리 서비스의 컨테이너 사양도, 기존 배너의 디자인 톤도, Admin API 형식도 모르니까요.
AI가 능력이 없는 게 아닙니다. 컨텍스트가 없는 겁니다.
이 문제를 SKILL.md 하나로 해결했습니다.
마크다운 파일에 역할 + 제약 + 도구를 정의하면, AI가 그 파일을 읽고 전문가처럼 동작합니다.
핵심은 세 가지입니다.
1. 역할(Role) — "배너 생성기"가 아니라 "AdBannerSlider 호환 배너를 만드는 프론트엔드 엔지니어"로 구체적으로 정의합니다.
2. 제약(Constraint) — 컨테이너 높이(160/180/200px), 색상 팔레트, WCAG 대비율, 네이밍 규칙을 명세합니다. 자유도가 높을수록 결과물의 일관성이 떨어집니다.
3. 도구(Tool) — 어떤 폴더를 읽고, 어디에 저장하고, 어떤 API를 호출하는지 정확히 지정합니다.
결과: `/create-banner 봄 분위기 테스트 홍보 배너` 한 줄이면 40초 안에 반응형 HTML 생성 + DB 등록까지 완료됩니다.
물론 완벽하진 않습니다. WCAG 대비율을 가끔 놓치고, 브라우저에서 실제로 렌더링해볼 수 없습니다. 하지만 체크리스트와 preview.html로 보완하면 실용적인 수준에서 충분히 동작합니다.
프롬프트는 그 순간만 삽니다. SKILL.md + 레퍼런스 폴더는 시스템으로 동작합니다.
규칙이 명확하고 반복적인 모든 작업은 이 패턴으로 자동화할 수 있습니다.
