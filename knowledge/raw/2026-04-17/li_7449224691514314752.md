## 원문

스스로 진화하는 하네스: Meta의 HYPERAGENTS
메타에서 스스로 하네스를 짜는 하네스 HYPERAGENTS를 공개했습니다. 이 녀석은 단순히 정답을 찾는 걸 넘어, '문제를 푸는 파이프라인과 학습법' 자체를 스스로 파이썬 코드로 짜면서 발명합니다.
흥미로운 부분은 메타 학습 능력의 전이입니다. 수학의 '수'자도 모르는 0점짜리 깡통 하네스가 있습니다. 여기에 생판 다른 도메인(논문 리뷰, 로봇 제어)에서 '공부하는 요령'만 터득하고, 단 50번의 자체 코드 수정 만에 63점짜리 수학 채점 하네스로 개조해 버립니다.
인간이 도메인 지식을 주입하지 않아도, 하네스가 스스로 깨우친 '학습하는 방법론' 자체가 다른 분야로 전이된다는 것을 증명한 것입니다. 이제 논문의 4가지 핵심을 정리하겠습니다.
1. 고정된 트랙을 부수다: 두 개의 자아를 합친 Hyperagent
기존 자가 발전 AI는 작업을 하는 '작업자'와 얘를 고치는 '평가자'가 분리되어 있었고, 평가 방식은 인간이 짠 고정 코드였습니다.
HYPERAGENTS는 이 둘을 하나의 프로그램으로 합쳤습니다.
비유하자면 단순히 수학 문제를 100개 푸는 학생이 아니라, 풀다가 "아, 나 계속 여기서 논리 오류 나네. 내 코드 안에 오답 노트(Persistent Memory)를 만들고, 편향 감지기(Bias Detector)를 추가해야겠다"라며 자기 뇌 구조를 스스로 개조하는 학생입니다.
2. 4가지 도메인의 파괴적인 성과, 그리고 창발된 인프라
연구진은 코딩, 논문 리뷰, 로봇 물리 제어, 수학 채점이라는 생판 다른 4곳에 얘를 던졌습니다.
- 코딩(버그 수정): 8.4% -> 26.7%
- 논문 심사(채택/거절): 0.0% -> 71.0%
- 로봇 점프력 상승: 0.060 -> 0.372
- 올림피아드 수학 채점: 56.1% -> 60.1%
코딩 성공률 26.7%면 현업에 당장 던져놓기엔 아직 부족합니다. 하지만 진짜 무서운 건 결과가 아니라 과정입니다.
인간이 개입하지 않았는데도 AI가 성능을 올리기 위해 '자체적인 다단계 검증 시스템'과 '과거 세대 평가 추적 로직'을 스스로 코딩해 넣으며 진화했습니다.
3. 창발적 메타 인프라 구축: AI가 스스로 발명한 공부 도구들
인간이 지시하지 않았음에도, AI는 오직 성능을 한계까지 끌어올리기 위해 아래와 같은 자체적인 인프라를 직접 코딩해 냈습니다.
- 성과 추적기 (Performance Tracking): 스스로 `PerformanceTracker` 클래스를 만들었습니다. 이전 세대들의 점수를 배열에 담고 이동 평균을 계산해, 자신의 코드 수정 방향이 전진하고 있는지 퇴행하고 있는지 수치화하여 추적합니다.
- 결과 편향 감지 (Bias Detection): 특정 예측 비율이 80%를 넘으면 편향으로 간주하는 방어 로직을 스스로 추가했습니다. 실제로 "99%를 합격시켜서 정확도가 박살 났다. 코드가 과교정(over-corrected)되었다"며 원인을 진단하고 코드를 롤백하여 정확도를 복구해 내는 모습을 보입니다.
- 연산 자원 분배 (Compute-aware Planning): 남은 반복 횟수(iterations_left)를 인식해 전략을 바꿉니다. "기회가 5번 이상 남았으면 코어 아키텍처를 근본적으로 갈아엎고, 2번 이하면 보수적으로 버그 수정만 하라"는 치밀한 분기 처리를 코딩해 넣었습니다.
- 영구 메모리 (Persistent Memory): `MemoryTool`을 만들어 타임스탬프와 함께 자신의 인사이트를 JSON 파일로 저장합니다. "55세대는 정확도는 높지만 너무 가혹했다. 64세대의 밸런스와 결합할 필요가 있다"며 과거의 삽질을 다음 세대가 반복하지 않도록 영구 기록합니다.
4. 8,860만 토큰의 끔찍한 가성비, 그리고 명확한 한계
물론 치명적인 한계도 명확합니다.
- 뇌(가중치)는 그대로: 파이썬 코드와 파이프라인 같은 외부 도구만 뜯어고칠 뿐, 기반이 되는 LLM 자체의 가중치를 업데이트하지는 못합니다. 공부법은 기가 막히게 바꿨지만 타고난 지능 자체를 바꾼 건 아닙니다.
- 가성비 최악: 에이전트 하나를 진화시키는 데 들어가는 연산량이 8,860만 토큰입니다. 실용성을 따지는 현업의 눈높이에서는 당장 실용성이 0에 수렴하는 컴퓨팅 자원 파쇄기입니다.
💡 마치며: 포스트 트레이닝의 미래
실용성은 떨어지지만, 이 논문이 증명한 가능성은 거대합니다. 단순히 정해진 문제를 푸는 도구에서, 자신을 스스로 진화시키는 '개방형 지능(Open-ended AI)'으로 넘어가는 패러다임 전환을 보여줬습니다.
이 '스스로 룰을 깎는 구조'를 RLVR(검증 가능한 보상 기반 강화학습) 파이프라인과 결합한다면 어떨까요? 단순히 프롬프트를 수정하는 수준을 넘어, 에이전트가 스스로 정교한 보상 함수를 설계하고 검증 데이터를 깎아내며 모델 가중치까지 스스로 업데이트하는 무한 루프가 열릴 수 있습니다.
