# SungJae Shim: AI 엔지니어 로드맵을 GitHub 커리큘럼과 DeepTutor를 결합해 한국어 튜터링·퀴즈·오류 피드백 워크플로우로 구현했다고 공유합니다.

**AI 요약**: AI 엔지니어 로드맵을 GitHub 커리큘럼과 DeepTutor를 결합해 한국어 튜터링·퀴즈·오류 피드백 워크플로우로 구현했다고 공유합니다.

**게시일**: 2026-04-20

**참신성 점수**: 62/100

**참신성 이유**: AI 엔지니어링 로드맵을 두 오픈소스(GitHub)로 엮고 DeepTutor로 ‘강의→한국어 설명→퀴즈→오류 수정’까지 실제 세팅을 제시해 실사용 관점의 구체성이 있으나, 기업 AX 전환보다는 개인 학습/역량 구축 중심이라 임팩트는 제한적입니다.

## 원문

지난주 MIT, 하버드, 스탠퍼드 무료 강의 정리 글을 올렸습니다.
감사하게도 많은 분들이 읽어주셨고,
DM과 댓글로 가장 많이 받은 질문은 이것이었습니다.
"AI 엔지니어가 되려면 뭐부터 시작해야 하나요?"
바로 답을 드리지 못했습니다.
검색하면 AI 로드맵 인포그래픽은 많이 나옵니다.
하지만 그 한 장을 보고 "오늘 당장 어디서부터" 시작할 수 있는 자료는 드물었습니다.
지난 한 주간 이 질문에 답이 될 만한 자료를 찾아 헤맸습니다.
그러다 두 개의 GitHub 리포지토리를 발견했고,
이 둘을 엮어서 쓰기 시작한 이후로 학습 생산성이 달라졌습니다.
첫 번째는 rohitg00/ai-engineering-from-scratch 입니다.
20개 Phase, 230개가 넘는 강의로 구성된 AI 엔지니어링 풀 커리큘럼입니다.
셋업과 수학, 머신러닝, 딥러닝, 컴퓨터 비전,
NLP, 음성 처리, Transformer, LLM 처음부터 만들기,
프롬프트 엔지니어링, RAG, 에이전트, MCP 서버,
멀티 에이전트, 오토노머스 시스템, 프로덕션 배포, 윤리,
그리고 캡스톤 프로젝트 5개까지.
Python, TypeScript, Rust, Julia 예제를 언어별로 나눠서 제공합니다.
개인이 만든 리포지토리라기엔 스케일이 상당합니다.
이 리포가 다른 강의 모음과 결정적으로 다른 점이 하나 있습니다.
슬로건이 "Learn it. Build it. Ship it for others"입니다.
강의 하나를 마치면 끝나는 구조가 아닙니다.
매 강의마다 프롬프트, 스킬, 에이전트, MCP 서버 중 하나를 만들게 설계되어 있습니다.
커리큘럼을 끝내면 outputs 폴더에
실무에서 바로 사용할 수 있는 도구가 누적되는 구조입니다.
한계도 있습니다.
아직 초기 단계라 Phase 0~2의 기초 부분은 대부분 채워져 있지만,
Phase 3 이후 고급 파트는 제목만 있는 강의도 있습니다.
실제 완성도는 전체의 4분의 1 수준입니다.
저는 이것이 오히려 장점이라고 판단했습니다.
"무엇을, 어떤 순서로 배워야 하는가"에 대한 지도로 활용하고,
비어 있는 부분은 MIT OCW 강의나 논문, 교재로 채우면 됩니다.
문제는 여기서부터입니다.
아무리 좋은 로드맵이 있어도,
결국은 영어 자료를 혼자 붙잡고 씨름해야 합니다.
막히면 그대로 끝입니다.
지난 MIT 글에서도 짚었던 오래된 벽입니다.
사실 이 벽을 해결할 도구에 대한 글을
9일 전에 한 번 올렸습니다.
"이제 문제집이 필요 없습니다"로 시작했던 DeepTutor 관련 포스팅입니다.
솔직하게 말씀드리면, MIT 글의 10분의 1도 반응이 나오지 않았습니다.
아이 교육이라는 앵글로만 풀어낸 것이 제 판단 미스였습니다.
이번 주에 ai-engineering-from-scratch를 돌려보면서
다시 DeepTutor를 꺼내 봤습니다.
이 도구의 진짜 타겟은 아이가 아니라
'혼자 공부하는 성인 전문직'이었습니다.
DeepTutor는 홍콩대 데이터 사이언스 랩에서 공개한 오픈소스입니다.
Star 1만 개가 넘는 검증된 프로젝트입니다.
단순 챗봇이 아닙니다.
교재, 논문, PDF를 통째로 업로드하면
여러 AI 에이전트가 역할을 나눠서 작동합니다.
문제 분석 에이전트, 풀이 에이전트,
검증 에이전트, 출처 추적 에이전트가 각자 움직입니다.
제가 이번 주에 실제로 구성한 세팅은 다음과 같습니다.
ai-engineering-from-scratch 리포의 강의 문서 전체를
DeepTutor의 Knowledge Base에 업로드했습니다.
마크다운 포맷이라 그대로 들어갑니다.
그다음 이렇게 요청했습니다.
"이 강의를 한국어로 설명해주세요.
이해했는지 확인할 수 있는 문제 3개를 내주세요.
틀린 부분이 있으면 그 부분만 다시 설명해주세요."
개인 교수 한 명이 생기는 경험이었습니다.
해외 석사 과정의 본질이 무엇인가 다시 생각하게 됐습니다.
체계적인 커리큘럼, 양질의 교재,
그리고 질문하면 답해주는 교수.
1억 원에 가까운 학비가 정당화되는 이유가 이 세 가지였습니다.
그 세 가지가 현재 시점에 모두 무료로,
개인 노트북 안에 존재합니다.
일주일간 직접 써보면서 정리한 활용 원칙 세 가지를 공유합니다.
1) 순차 학습이 아닌, 문제 기반 학습으로 접근하세요.
Phase 0부터 20까지 순서대로 학습하는 것은
전통적인 대학 커리큘럼 방식입니다.
에이전트를 만들어야 하면 Phase 14부터 시작하고,
RAG 시스템이 필요하면 Phase 11부터 들어가세요.
수학 기초는 막히는 순간에 돌아와서 채우면 됩니다.
"처음부터 끝까지"가 아니라 "필요한 순간에"가
지금 시대의 학습 방식입니다.
2) 도구의 역할을 분리하세요.
학습 자료 이해, 문제 풀이, 개념 검증은 DeepTutor.
실제 코드 작성과 디버깅은 Claude Code 또는 Cursor.
한 도구로 모든 것을 처리하려 하면 두 영역 모두 애매해집니다.
3) 모든 강의를 "산출물"로 마감하세요.
RAG 강의는 회사 문서 챗봇으로.
에이전트 강의는 반복 업무 자동화 도구로.
MCP 강의는 평소 사용하는 SaaS 연동으로.
그날 적용해보지 않은 지식은 일주일 안에 사라집니다.
직접 겪으면서 체득한 원칙입니다.
한계도 분명히 말씀드립니다.
엔지니어링 리포는 고급 파트가 아직 비어 있습니다.
DeepTutor는 OpenAI API 키가 필요하고, 약간의 토큰 비용이 발생합니다.
공식 한국어 UI도 아직 지원되지 않습니다.
다만 한국어로 질문하면 한국어로 답변은 잘 나옵니다.
그럼에도 5년 전이었다면 상상할 수 없었던 학습 환경인 것은 분명합니다.
지난 MIT 글에서 드렸던 말씀을 다시 한 번 드리며 마무리합니다.
강의는 무료입니다.
AI 튜터는 24시간 대기 중입니다.
로드맵까지 누군가가 정리해두었습니다.
이제 진짜 남은 것은 하나입니다.
오늘 1강이라도 열어보셨습니까.
GitHub 링크는 댓글에 남기겠습니다.
