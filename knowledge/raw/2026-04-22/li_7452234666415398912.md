## 원문

슬라이드에서 영상까지 콘텐츠 엔진이 완성되다
저번 포스트에서는 clone-slide 하네스 결과물을 보여드렸습니다.
PDF 슬라이드를 html, png, pptx 그대로 복제할 수 있는 하네스였죠.
그 다음 질문이 자연스럽게 따라왔습니다.
"그럼 영상으로도 뽑을 수 있나?"
결론부터 말하면, 됩니다.
슬라이드 제작부터 영상 전환까지, 콘텐츠 엔진을 전부 구축했습니다.
또 하나의 '딸깍' 매직입니다.
슬라이드의 영상 전환.
Hyperframes 의 HTML + CSS + GSAP 조합이 이걸 가능하게 만들었습니다.
github repo: https://lnkd.in/gNsc5bRS
슬라이드에 들어간 애니메이션과 전환이 그대로 mp4 로 떨어집니다.
이 스킬의 built-in template 이 가장 잘 작동하지만, 원하는 방식이 있다면 미리 세팅해두면 됩니다. 한 번 세팅해두면 그 이후로는 정말 '딸깍'입니다.
가장 큰 이슈 하나.
GSAP 렌더링 이슈로 페이지 간 전환이 너무 빠르게 지나가거나 내용이 사라지는 문제가 있습니다. 내용을 확인할 수 없으면 영상이 의미가 없죠.
저는 'hold window on the actual page DOM' 이라는 키워드로 해결했습니다. 각 페이지를 충분히 잡아두고 전환만 부드럽게 연결하는 방식입니다.
정리하면 이런 흐름입니다.
PDF/이미지 → clone-slide 로 HTML 슬라이드 복제
HTML 슬라이드 → Hyperframes 로 영상 전환
중간 어디서든 pptx 로도 추출 가능
여기서 개인적인 얘기.
요즘 제 개발 환경입니다.
