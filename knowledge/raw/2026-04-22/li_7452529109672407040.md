## 원문

Everyone reaches for fine-tuning a AI Model. Almost no one needs to.
Teams spend 6 weeks on training runs that a better system prompt would have solved in 2 hours.
Before you rent a GPU, run the 3-layer rule:
🔴 Layer 01 — Prompt engineering + AI Evals
Rewrite the system prompt. Add clear examples. Measure what's failing. Most problems die here.
🔵 Layer 02 — Add RAG
If the model is hallucinating because it lacks context — don't bake knowledge into weights. Retrieve it.
⚠️ Layer 03 — Fine-tune
Only when the failure is behavioral. Wrong tone. Persistent classification errors. Format prompting can't fix. Last resort, not first instinct.
Teams almost always reach for Layer 03 the moment prompting gets "annoying."
That's not an LLM problem. That's an impatience problem.
Majority of the issues (if not all) get solved at Layer 01 (Prompt Engineering).
