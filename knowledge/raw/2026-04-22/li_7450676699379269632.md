## 원문

🎁 Claude Opus 4.7… here we go!
Big changes to how the model behaves.
It’s designed to reason more, reaches for tools less, follows instructions more literally, and checks its own work before reporting back.
That changes how you prompt and how much oversight the work actually needs.
💡What stands out:
-Coding performance jumped significantly. SWE-bench Pro went from 53.4 to 64.2. Cursor’s benchmark cleared 70% vs 58% on Opus 4.6. That means: pushing more production tasks through it.
-Fewer tool calls, more reasoning is usually better. Be more explicit in your prompts about when you need it to use specific tools, and leverage the new “xhigh” effort level for heavier agentic workflows.
-Visual resolution tripled. Big unlock for computer-use agents, visual analysis and document extraction.
-Session memory improved. Less re-prompting, more continuity in long-running agentic work.
-Code review got a dedicated tool. The new /ultrareview command in Claude Code flags bugs and design issues like a careful human reviewer would.
-Cyber safeguards are now in production. This is the first model shipping with Anthropic’s Project Glasswing protections.
-Token consumption may increase. The updated tokeniser can produce up to 1.35x more tokens on the same input. When shifting to this model on heavy production tasks, monitor carefully.
-Same pricing as Opus 4.6, so I assume the plan is to discontinue the former one. Available everywhere: Claude Code, API, Bedrock, Vertex AI, Foundry.
-Adaptive reasoning replaces manual thinking controls, the model decides how hard to think. Works well for code and maths, but early reports suggest it may underestimate analysis, writing, and research tasks. I’ll test this myself to see if the difference is material vs Opus 4.6.
-Context discipline still wins. The model amplifies whatever you feed it. Your instruction quality, context engineering, and observability layer still determine whether capability translates into outcomes.
This release feels meaningful.
A model designed to reason more before acting, check its own work, and cut friction on multi-step tasks so builders can stay in flow.
That’s a different kind of building partner. One that challenges more and avoids agreeing blindly on complex decisions.
Something tells me I won’t sleep much over the coming days.
