## 원문

AI로 1인 게임 스튜디오를 세울 수 있는 시대, 수익화 채널까지 열려 있습니다
2025년 Steam 전체 매출 177억 달러 중 25%가 인디 게임이었습니다.
45억 달러, 한화 6조 원 규모의 시장입니다.
올해 3월, 프로그래밍을 독학한 1인 개발자가 타워 디펜스 게임을 Steam에 출시했습니다. 첫 주 매출 24만 5천 달러. 약 3억 3천만 원. 4년간 혼자 만든 게임이었습니다.
Among Us는 5만 달러 미만의 예산으로 1억 달러 이상의 매출을 기록했고, Stardew Valley 역시 1인 개발 프로젝트였습니다.
물론 현실적인 수치도 함께 봐야 합니다.
2025년 Steam에 출시된 게임은 약 2만 개였지만, 매출 중간값은 249달러에 불과합니다. 100만 달러를 넘긴 타이틀은 약 300개. 시장은 크지만 대부분의 게임은 발견되지 못합니다.
그럼에도 주목하는 이유는 구조적 변화가 일어나고 있기 때문입니다.
게임 개발의 진입 장벽은 늘 “기술력”보다 “조직”이었습니다. 기획, 개발, QA, 밸런싱, 사운드를 혼자 감당하는 것이 물리적으로 불가능했기 때문입니다.
최근 공개된 Claude Code Game Studios라는 오픈소스 프로젝트는 이 문제에 정면으로 접근합니다.
Claude Code 위에서 48개의 AI 에이전트가 게임 스튜디오의 조직 구조를 그대로 재현합니다.
3 Tier 구조:
Tier 1 - 크리에이티브 디렉터, 테크니컬 디렉터, 프로듀서
Tier 2 - 게임 디자이너, 리드 프로그래머, QA 리드
Tier 3 - 게임플레이 프로그래머, 이코노미 디자이너, 사운드 디자이너 등
디렉터급은 비전을 수호하고, 리드급은 도메인을 관리하며, 실무급이 실제 작업을 수행합니다. 도메인 간 충돌이 발생하면 상위 에이전트로 에스컬레이션되는 구조까지 갖추고 있습니다.
핵심은 이것이 자동 생성기가 아니라는 점입니다.
모든 에이전트가 먼저 질문하고, 2-4개의 선택지를 제시하며, 최종 결정과 승인은 반드시 사람이 합니다. 기획은 사람이 하고, 실행은 48명의 AI 팀이 수행하는 “구조화된 협업” 모델입니다.
/brainstorm으로 아이디어를 발산하고, /sprint-plan으로 개발 계획을 수립하고, /code-review, /balance-check, /release-checklist까지. Godot, Unity, Unreal Engine을 모두 지원합니다.
GitHub 스타 7,200개 이상. 포크 1,000개 이상.
수익화 채널도 이미 갖춰져 있습니다.
PC 게임은 Steam과 Epic Games Store, 모바일은 Google Play와 App Store, 프로토타입 테스트는 itch.io, 웹 게임은 자체 결제 연동까지. 등록비는 플랫폼별로 몇만 원 수준이고, 인앱결제와 광고 수익 모델도 엔진 단에서 바로 연동됩니다. 유통 인프라는 이미 완성되어 있습니다.
물론 한계도 명확합니다.
AAA급 타이틀은 불가능하며, Claude Code 유료 구독이 전제 조건입니다. 3D 에셋을 AI가 직접 생성해주는 것도 아닙니다.
그러나 턴제 전략, 텍스트 시뮬레이션, 로그라이크, 타워 디펜스, 퍼즐 장르는 기획 밀도가 품질을 결정하는 영역입니다. 그리고 이 장르들이 실제로 인디 시장에서 수익을 만들어내는 카테고리이기도 합니다.
비즈니스 관점에서의 시사점은 이렇습니다.
게임 개발의 병목이 “조직 구성”에서 “기획력 + 마케팅”으로 이동하고 있습니다. 도메인 전문성을 가진 개인이 AI를 실행 도구로 활용할 때, 과거에는 팀 단위로만 가능했던 결과물을 만들 수 있는 구조가 형성되고 있습니다.
이것은 게임 산업에만 국한된 이야기가 아닙니다. “아이디어는 있지만 실행할 조직이 없다”는 문제를 AI가 해결하기 시작하면, 콘텐츠, 교육, 시뮬레이션 등 기획 집약적 산업 전반에서 동일한 변화가 일어날 것입니다.
여러분의 산업에서 “조직의 벽” 때문에 실행하지 못했던 프로젝트가 있으신가요?
