- **AI 자동화 해고의 수요 붕괴 트랩**: Nash Equilibrium 하에서 개별 기업의 합리적 선택이 집단적 수요 붕괴를 초래하며, 피구적 자동화 세금이 이론적 해법이나 지정학적 현실에서는 작동하기 어렵다 (염기웅)
- **SFT 시 스타일 충돌이 고품질 데이터의 역설을 만든다**: Teacher 데이터 품질보다 Student 모델의 스타일 유지가 성능에 더 크게 기여한다 (TESSY 논문, 염기웅)

## 주제별 정리

### AI 거버넌스 & 검토 체계

**사실:**
- 황현태는 AI 초안 속도 증가로 리더/관리자의 검토 병목이 "아주아주 무지무지 크게" 생긴다고 밝혔다
- LLM-as-Judge가 90% 수준의 검토를 도울 수 있으나 마지막 판단은 여전히 인간에게 귀착된다고 언급
- Nichelle Maynard-Elliott는 Semafor World Economy에서 "AI 역량이 거버넌스를 앞지르고 있으며, 실패 시 이사회가 책임을 물어야 한다"는 컨센서스를 보고했다

**의견:**
- 황현태: AI 시대에 완성도 개념과 끝까지 책임지는 태도가 더욱 중요해진다
- Marty Haak: AI를 방치하면 messy output·중복 작업·추가 리뷰 루프가 발생하며, 소유권·품질 기준·시스템 연동이 선행되어야 한다
- Nichelle: AI를 기존 프로세스 4단계에 얹는 것은 전환이 아닌 '오버레이'이며, "워크플로우 수술(Workflow surgery)"이 진정한 전환이다

**AX 시사점:**
- AI 도입 조직은 생산량 증가에 따른 검토 부담 재설계를 선행해야 하며, 역할 분리(초안 생성 vs. 완성도 책임)를 명확히 해야 한다
- 이사회·경영진 수준에서 AI 실패 책임 소재를 사전 정의하는 거버넌스 체계가 필요하다

### AI SLA & 인프라 신뢰성

**사실:**
- 조여준에 따르면 Claude.ai 업타임 98.19% = 연 158시간(6.6일) 중단, 월 평균 13시간 장애
- 엔터프라이즈 SaaS 기준선은 99.9%, AWS/Google Cloud는 99.99%를 제공한다
- Claude API 활용 시 업타임이 98.98%로 상승한다

**의견:**
- 조여준: 상업적 Agentic workflow는 API를 써야 하며, 이러한 격차가 계속되면 많은 workflow가 중국 오픈소스로 이동할 것이다

**AX 시사점:**
- 24시간 에이전트 운영 설계 시 LLM 제공자의 SLA를 비용과 함께 반드시 평가 항목에 포함해야 한다
- Claude.ai 웹 의존 워크플로우는 운영 리스크가 존재하며, API 기반 아키텍처로 전환이 권장된다

### AI 자동화와 거시경제 리스크

**사실:**
- UPenn·Boston University 논문 <The AI Layoff Trap>은 AI 자동화 경쟁이 Nash Equilibrium에서 수요 붕괴를 수학적으로 증명했다
- UBI는 기업의 해고 인센티브를 제거하지 못하고, 강성 노조 파업은 글로벌 수요 붕괴를 막지 못한다
- 논문의 이론적 해법은 피구적 자동화 세금(Pigouvian tax)이나, 미중 패권 경쟁 맥락에서 현실성이 낮다

**의견:**
- 염기웅: 한국의 법적 해고 제한·강성 노조가 역설적으로 AI 자동화 속도에 브레이크를 걸어 수요 절벽을 늦출 수 있다
- Poorna Reddy: 에이전트 자동화가 먼저 흡수하는 것은 분해 가능성이 높은 업무(boilerplate, test scaffolding, API integration)이며, 남은 잔여 복잡성은 축소된 팀이 감당하지 못할 수 있다

**AX 시사점:**
- AX 전략에서 인력 전환(reskilling) 계획 없는 자동화는 단기 비용 절감 이후 운영 리스크로 귀환한다
- 기업 AX 설계 시 agent objective function 정의·출력 검증·정렬 유지 역량을 의도적으로 팀 내에 구축해야 한다

### AI 모델 학습 전략 (기술)

**사실:**
- TESSY 논문: Teacher 데이터만으로 Qwen3-8B를 SFT하면 OJBench -10.02%, LiveCodeBench-V5 -13.77% 성능 하락
- TESSY(교사=추론 토큰, 학생=스타일 토큰 분리)로 학습 시 각각 +11.25%, +6.68% 성능 상승
- ByteDance Ouro/LoopRPT: 1.4B 모델이 루프 재귀 구조(최대 4회)와 강화학습 기반 사전학습으로 대형 모델을 수학·코딩에서 능가

**의견:**
- 염기웅: 데이터 절대 품질보다 학생 모델 스타일 유지가 SFT에서 더 중요하다
- 루프형 언어모델은 Chain-of-Thought 토큰 출력 없이 잠재 공간에서 다단계 추론을 수행해 효율성을 높인다

**AX 시사점:**
- 소형 모델 커스터마이징 시 Teacher 데이터 품질보다 배포 모델과의 스타일 정합성이 핵심 변수이다
- 기업 내부 도메인 파인튜닝 프로젝트에서 TESSY 패턴(역할 분리 데이터 생성)을 참조할 수 있다

### AI POC → 운영화 전환

**사실:**
- Pankaj Zanke: AI POC 실패 원인으로 불명확한 KPI, 지저분한 실데이터, 프로덕션 아키텍처 부재, 거버넌스 무시, 오너십 부족을 열거했다

**의견:**
- AI 성공 기준이 "만들 수 있는가?"에서 "운영화할 수 있는가?"로 이동해야 한다
- 황현태: 리더가 직접 "생산성 10배"를 체득한 AI-Native Builder가 된 뒤에야 조직 재설계가 의미 있다 (꼭짓점 AX)

**AX 시사점:**
- POC와 운영 사이에 있는 데이터 품질·아키텍처·거버넌스 갭을 미리 설계하는 "운영화 체크리스트"가 프로젝트 초기부터 필요하다

## 트렌드 & 시사점

1. **검토 부담의 구조적 재설계 수요 급증**: AI 초안 생산이 일반화되면서 조직 내 품질 게이트(Quality Gate) 역할이 새로운 병목으로 부상. "LLM-as-Judge + 인간 최종 검토" 하이브리드 레이어 설계가 AX 프로젝트의 핵심 과제로 이동하고 있다.

2. **API vs. 웹 UI 이분법이 상업적 Agentic 운영의 기준선이 됨**: Claude SLA 사례처럼 LLM 제공자 선택이 단순 성능이 아닌 가용성·비용·지정학적 리스크(중국 오픈소스 대안)를 포함한 다차원 평가 문제가 됐다.

3. **'스타일 충돌' 개념이 기업 파인튜닝 실패 진단에 적용 가능**: Teacher-Student 데이터 증류의 실패가 모델 품질이 아닌 분포 불일치에서 비롯된다는 TESSY 발견은, 기업 도메인 파인튜닝 실패 사례를 재해석하는 프레임을 제공한다.

4. **이사회 수준 AI 거버넌스 의제화 가속**: Semafor 사례처럼 AI 실패 책임 소재가 경영진을 넘어 이사회로 올라오고 있으며, 에너지 인프라·규제 파편화가 신규 리스크 항목으로 등장하고 있다.
